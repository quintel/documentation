#### New direct emissions method
* A new method for emissions accounting has been added to the model. This method determines direct greenhouse gas emissions at the point where emissions are released.
* The results of the direct emissions method can be obtained via detailed <a href="https://energytransitionmodel.com/scenario/data/data_export/yearly-direct-greenhouse-gas-emissions" target="_blank">data exports</a>.
* **Note:** the direct emissions method is currently in beta release.
* Read more about the beta release in the <a href="https://docs.energytransitionmodel.com/main/direct-emissions" target="_blank">documentation</a>.

#### PyETM 2.0 release
* [PyETM](https://docs.energytransitionmodel.com/main/PyETM/introduction) version 2.0 is officially released and available via [PyPi](https://pypi.org/project/pyetm/).
* To learn more about the Python connector for the ETM check out the documentation [here](https://quintel.github.io/pyetm/).
* PyETM will replace the [scenario-tools](https://docs.energytransitionmodel.com/main/scenario-tools/introduction), which will be retired in the next release. If you need help switching to from scenario-tools to PyETM, please <a href="https://my.energytransitionmodel.com/contact" target="_blank">get in touch</a>.

#### Other new features
* New data exports have been added, allowing users to retreive peak capacities for electricity, hydrogen, network gas, and district heating. Existing hourly profile exports have been renamed for clarity, see the [documentation](/api/exports). Backwards compatibility with the previous names is maintained.
