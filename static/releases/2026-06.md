#### New features

##### New direct emissions method
A new method for emissions accounting has been added to the model. This method determines direct greenhouse gas emissions at the point where emissions are released.

The results of the direct emissions method can be obtained via detailed <a href="https://energytransitionmodel.com/scenario/data/data_export/yearly-direct-greenhouse-gas-emissions" target="_blank">data exports</a>. These data exports contain the start year and future year direct emission results for all technologies in the model. Documentation about the method can be found <a href="https://docs.energytransitionmodel.com/main/direct-emissions" target="_blank">here</a>.

**Note: the direct emissions method is currently in beta release.** This means that it is being implemented incrementally and is not yet completely available for regional (Dutch) datasets. Read more about this in the above-mentioned documentation.

##### PyETM 2.0 Release
[PyETM](https://docs.energytransitionmodel.com/main/PyETM/introduction) version 2.0 is officially released and available [via pypi](https://pypi.org/project/pyetm/). To learn more about the Python connector for the ETM check out the documentation [here](https://quintel.github.io/pyetm/). PyETM will replace the [scenario-tools](https://docs.energytransitionmodel.com/main/scenario-tools/introduction), which will be retired with the next deploy. If you need help switching to from scenario-tools to PyETM, reach out to info@energytransitionmodel.com.

##### Data exports for capacities
Added new CSV exports for participant capacities, allowing users to download installed and peak capacities for electricity, hydrogen, network gas, and heat networks. Existing hourly curve exports have also been renamed to more clearly distinguish profile and capacity exports, while maintaining backwards compatibility with existing routes. [Learn more here.](/api/exports)
